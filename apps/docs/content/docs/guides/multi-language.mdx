Turborepo can run tasks across multiple language ecosystems by using [`workspaceProviders`](/docs/reference/configuration#workspaceproviders) in your root `turbo.json`.

Available provider IDs:

- `node`
- `cargo`
- `uv`

## Quick start

Add providers at the root:

```json title="./turbo.json"
{
  "workspaceProviders": ["node", "cargo", "uv"],
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", "target/**", "build/**"]
    },
    "test": {
      "dependsOn": ["^build"],
      "outputs": []
    }
  }
}
```

Then run:

```bash title="Terminal"
turbo run build test
```

Turborepo will resolve commands per workspace manifest:

- Node (`package.json`) → package scripts
- Cargo (`Cargo.toml`) → inferred cargo commands (for example `cargo build`, `cargo test`)
- uv (`pyproject.toml`) → inferred uv commands (for example `uv build`, `uv run pytest`)

Use dry-run to inspect what Turborepo resolved:

```bash title="Terminal"
turbo run build --dry=json
```

## Example repository layout

```text
.
├── turbo.json
├── package.json              # node workspace root
├── Cargo.toml                # cargo workspace root
├── pyproject.toml            # uv workspace root
├── apps/
│   └── web/package.json
├── crates/
│   ├── rust-util/Cargo.toml
│   └── rust-app/Cargo.toml
└── packages/
    ├── py-util/pyproject.toml
    └── py-app/pyproject.toml
```

## Command precedence

If you define an explicit command on a task, it takes priority over inferred commands:

```json title="./turbo.json"
{
  "tasks": {
    "build": {
      "command": "echo custom build command"
    }
  }
}
```

See [`command`](/docs/reference/configuration#command) for details.

## Dependency relationships

Cross-workspace task ordering still uses normal task graph rules:

- `dependsOn: ["^build"]` runs dependency packages first
- internal dependencies are inferred from each provider's manifest conventions
  (for example Cargo path/workspace dependencies, uv workspace sources)

## Hands-on examples

You can bootstrap provider examples directly:

```bash title="Terminal"
npx create-turbo@latest -e with-cargo
npx create-turbo@latest -e with-uv
npx create-turbo@latest -e with-workspace-providers
```
