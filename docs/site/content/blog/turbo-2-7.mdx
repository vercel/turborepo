# Turborepo 2.7

<Date>Friday, December 19th, 2025</Date>

<Authors authors={['anthonyshew', 'tomknickman']} />

Turborepo 2.7 is now available, including new features like:

- [**Devtools**](#devtools): Visualize your Package Graph and Task Graph
- [**Composable configuration**](#compose-configuration-with-extends): Extend Package Configurations from other configurations
- [**New Turborepo rule in Biome**](#new-biome-rule-noundeclaredenvvars): Lint with Biome to find unsafe usage of environment variables
- [**Yarn 4 catalogs**](#yarn-4-catalogs): Centralized dependency management for Yarn

Upgrade today by running `npx @turbo/codemod migrate` or get started with `npx create-turbo@latest`.

## Devtools

We’ve shipped a brand new experience at [turborepo.com/devtools](http://turborepo.com/devtools). Starting with this version of Turborepo, you can run `turbo devtools` to visually explore your Package Graph and Task Graph.

The graphs will hot-reload as you make changes to your repository, and help you to answer questions like:

- Which packages will miss cache when I change my utilities package?
- What tasks does this task depend on, directly and indirectly?
- What packages does _this specific package_ depend on that makes it miss cache so often?

And much, much more. Visit [turborepo.com/devtools](http://turborepo.com/devtools) to get started, or [learn more in the documentation](/docs/reference/devtools).

## Compose configuration with `extends`

We’ve expanded the capabilities of [Package Configurations](/docs/reference/package-configurations) so you can write your configuration once, and share it to anywhere in your repository.

### A more powerful `extends` key

Previously, you could only extend from your root `turbo.json` when you used the `extends` key. Now, you can reference `turbo.json` from any other package in your workspace by the package name:

<Tabs items={["./apps/docs/turbo.json", "./apps/web/turbo.json"]}>

<Tab value="./apps/docs/turbo.json">

```json title="./apps/docs/turbo.json"
// Inherits the "build" and "prepare-build" tasks from apps/web/turbo.json
{
  "extends": ["//", "web"]
}
```

</Tab>

<Tab value="./apps/web/turbo.json">

```json title="./apps/web/turbo.json"
{
  "tasks": {
    "build": {
      "dependsOn": ["prepare-build"],
      "outputs": ["dist/**"]
    },
    "prepare-build": {}
  }
}
```

</Tab>

</Tabs>

[Visit the docs to learn more.](/docs/reference/package-configurations#extending-arrays-with-turbo_extends)

### Extending array configurations

Additionally, we’re introducing a new keyword: `$TURBO_EXTENDS$`.

```json title="./apps/web/turbo.json"
{
  "extends": ["//"],
  "tasks": {
    "build": {
      // Inherits "dist/**" from root AND adds ".next/**"
      "outputs": ["$TURBO_EXTENDS$", ".next/**"]
    }
  }
}
```

Previously, you could only overwrite array configurations completely. But now, in the example above, you can use `$TURBO_EXTENDS$` to append to an existing array. This gives you a greater degree of control so you can choose to completely overwrite or extend from existing configuration.

[Visit the docs to learn more.](/docs/reference/package-configurations#extending-arrays-with-turbo_extends)

## New Biome rule: `noUndeclaredEnvVars`

Turborepo users love going fast, so its no surprise they enjoy using [Biome](https://biomejs.dev/) for linting and formatting. However, until today, the only linting support we had for finding unsafe environment variable usage was in ESLint, with [our `eslint-plugin-turbo` plugin](/docs/reference/eslint-plugin-turbo).

In [Biome 2.3.10+](https://github.com/biomejs/biome/releases/tag/%40biomejs%2Fbiome%402.3.10), you can find environment variables that can cause unexpected cache hits faster with [the new Turborepo domain](https://biomejs.dev/linter/domains/#turborepo) in Biome.

Biome will read your dependencies in your repository, and activate the Turborepo domain automatically. The `noUndeclaredEnvVars` rule is currently in the “nursery” group, so must be activated manually:

```json title="biome.json"
{
  "linter": {
    "rules": {
      "nursery": {
        "noUndeclaredEnvVars": "error"
      }
    }
  }
}
```

Once the rule is promoted out of the nursery group in a future release of Biome, the rule will be automatically activated for Turborepo projects, requiring zero-configuration. To learn more about Biome, visit [the Biome documentation](https://biomejs.dev/).

## Yarn 4 catalogs

[Yarn’s 4.10.0 release](https://github.com/yarnpkg/berry/releases/tag/%40yarnpkg%2Fcli%2F4.10.0) introduced “catalogs”, inspired by [pnpm’s catalogs feature](https://pnpm.io/catalogs). Catalogs allow you to define a specific version of a dependency once, and reuse that version as a constant in the rest of your repository.

```yaml title=".yarnrc.yml"
catalog:
  react: ^19.2.3
  lodash: ^4.17.21
```

We’ve updated our lockfile parser to work with this new Yarn feature, ensuring our smart lockfile detection doesn’t cause global invalidations in your repository. Only the packages and tasks that are affected by the change to your catalogs will see cache misses, improving your cache hit rates.

[Visit our Support Policy](/docs/support-policy#package-managers) to learn more about our package manager support, or learn more about catalogs in [the Yarn documentation](https://yarnpkg.com/features/catalogs).
