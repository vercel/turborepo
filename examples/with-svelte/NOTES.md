# TypeScript Config Notes

As the SvelteKit generates tsconfig files but the repo provides a general config, that may be overridden by each app or
package, this document outlines the relationships of tsconfig files to make changes easier to apply.

## Relationships

```
packages/typescript-config/svelte.json   ← shared base
        ↑                    ↑                    ↑
apps/docs/tsconfig.json  apps/web/tsconfig.json  packages/ui/tsconfig.json
```

Each consumer extends two configs in order:

```json
{
	"extends": ["./.svelte-kit/tsconfig.json", "@repo/typescript-config/svelte.json"]
}
```

- **`.svelte-kit/tsconfig.json`** is auto-generated by SvelteKit and provides SvelteKit-specific settings: `$lib`/`$app`
  path aliases, `rootDirs`, and `noEmit: true` (type-checking only — Vite handles transpilation and source maps).
- **`@repo/typescript-config/svelte.json`** is the shared base that applies project-wide compiler settings (`strict`,
  `moduleResolution: bundler`, `verbatimModuleSyntax`, etc.). Later entries in `extends` win, so these settings take
  precedence over the generated config where they overlap.
- Each **consumer config** (`apps/docs`, `apps/web`, `packages/ui`) only overrides `rootDir` — everything else inherits
  from the base.

## `rootDir` and `outDir`

Both are path-sensitive settings that TypeScript resolves relative to the config file they appear in, so neither belongs
in the shared base config.

`rootDir` is set in each consumer because it is structurally meaningful even with `noEmit: true`: TypeScript uses it to
validate that no source file is imported from outside the declared root, catching accidental cross-package imports.
Omitting it would allow files outside `src/` to silently enter the compilation.

`outDir` is absent from all configs because the base config sets `noEmit: true` — TypeScript never writes output, so
`outDir` has no effect. Vite handles transpilation and source maps at build time. If `noEmit` were ever removed from the
base config, `outDir` would need to be added back to each consumer.
