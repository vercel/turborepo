# Turborepo Rsbuild module federation SSR starter

This is a community-maintained example. If you experience a problem, please submit a pull request with a fix. GitHub Issues will be closed.

## Using this example

Run the following command:

```sh
npx create-turbo@latest -e with-rsbuild-module-federation-ssr
```

## What's inside?

This Turborepo includes the following packages/apps:

### Apps and Packages

- `react-host`: an [Rsbuild](https://rsbuild.rs/) React host app with server-side rendering that consumes the remote app
- `react-remote`: an [Rsbuild](https://rsbuild.rs/) React remote app that exposes `./remote-app`
- `@mf-rsbuild-ssr-example/shared-ui`: a stub React component library shared by both applications

Each package/app is 100% [TypeScript](https://www.typescriptlang.org/).

### Utilities

This Turborepo has some additional tools already setup for you:

- [TypeScript](https://www.typescriptlang.org/) for static type checking
- [Rsbuild](https://rsbuild.rs/) for local development and production builds
- [Module Federation](https://module-federation.io/) for runtime composition between the host and remote applications
- [Express](https://expressjs.com/) custom server for SSR in development and preview

### Build

To build all apps and packages, run the following command:

```
turbo build
```

### Develop

Visit `localhost:3000` to interact with your SSR host and remote applications. The host app uses a custom Express server on port `3000` and the remote app runs on port `3001`.

The host resolves the remote through `http://localhost:3001/mf-manifest.json`, generated by `@module-federation/rsbuild-plugin`. The host renders HTML on the server and hydrates on the client.

To develop all apps and packages, run the following command:

```
turbo dev
```

### Preview

After building, preview the production SSR output:

```
turbo preview
```
