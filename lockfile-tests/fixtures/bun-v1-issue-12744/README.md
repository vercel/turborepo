# Turbo + Bun Pruned Lockfile Repro

Reproduction for a Bun lockfile generated by `turbo prune --docker` failing under `bun install --frozen-lockfile` in a Docker build.

This repo intentionally keeps source code minimal while preserving the relevant workspace `package.json` dependency graph from the original project. The target image is a Prisma migration service.

## Reproduce

```sh
docker build --no-cache -t turbo-bun-prune-frozen-repro -f apps/migrations/Dockerfile .
```

Expected result: the Docker build fails in the installer stage at:

```sh
bun install --frozen-lockfile --ignore-scripts
```

Observed failure with `oven/bun:1.3.13-alpine` and `turbo@2.9.7-canary.13`:

```text
error: Failed to resolve prod dependency 'eventemitter3' for package 'recharts'
InvalidPackageInfo: failed to parse lockfile: 'bun.lock'
warn: Ignoring lockfile
error: lockfile had changes, but lockfile is frozen
```

The failure happens before Prisma generation. The Prisma package is included to match a migration-service Docker image shape, while `--ignore-scripts` prevents `postinstall` from running before the pruned source is copied.

## Control

The root lockfile itself is valid:

```sh
docker run --rm -v "$PWD:/work" -w /work oven/bun:1.3.13-alpine bun install --frozen-lockfile --ignore-scripts
```

The failure only appears after `turbo prune --docker` generates `out/json/bun.lock`.
